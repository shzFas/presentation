About React js library

A bit of history

React

  • Frontend and JS library to build user interfaces
  • Makes it easier to create responsive WP
  • Dev by Facebook

Fundamentals

1) Component

  • Fundamental building block of React
  • Lets you split up the UI into indepent, reusable pieces, and think about each piece in isolation
  • Conceptually they are like JS functions
    • Accept arbitrary inputs calles "props"
    • Return React elements that describe what should appear on the screen

Component Example

2) Props

  • Passed down to component from parent component
  • Should be immutable or "pure"
    • Read-Only, never modified
  • All React components must act like pure functions with respect to their props

Props

3) State

  • Represents the internal state of the component
  • When a components state data changes, the component is re-rendered

4) Virtual DOM

  • Document Object Model(DOM)
  • The real DOM is the structure of a webpage

Virtual DOM

  • DOM maniulatation is the heart of the modern, interactive web
  • Unfortunately, it's a lot slower than most JS operations
  • Slowness is made worse by the fact that most JS frameworks update the DOM much more than they have

Virtual DOM

  • To address this inefficiency, React popularized the virtual DOM for efficient re-rendering of the actual DOM
  • The virtual DOM is a lightweight copy of the real DOM
  • React aims to re-render the virtual tree only when state changes
  • Once the virtual DOM has updated, React compares the virtual DOM with a virtual DOM snapshot before the state change
  • It then compares which obj have been changed and only updates those obj on the real DOM

5) JSX

  • JS extension
  • Allows us to combine js and html

Pros

  • Fast and efficient thought the use of a virtual DOM
  • Makes writing JS easier with JSX
  • Easy to read and understand code
  • React dev tools
    • Chrome(Mozila) extension makes debugging easier

Cons

  • React is only the view layer
  • Learning curve
